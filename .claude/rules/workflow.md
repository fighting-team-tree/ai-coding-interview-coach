# Workflow Rules (Claude Code)

## 실질적 작업 4단계

멀티파일 편집 · 디렉토리 구조 변경 · 제안서 수정 · 아키텍처 변경 · 커밋 준비가 해당됨.

1. **탐색** — 현재 상태 파악, 관련 파일 읽기
2. **브리핑** — 변경 파일 · 예상 결과 · 트레이드오프를 텍스트로 제시
3. **승인** — `AskUserQuestion` 도구로 선택지를 제시하고 명시적 동의 확인
4. **실행** — 합의한 범위만 기계적으로 수행

계획과 실행은 반드시 **별도 턴**으로 분리.
사용자가 "진행해줘"라고 해도 3단계(AskUserQuestion)는 생략 불가.

## 승인 불필요 (즉시 실행)

- 파일 읽기 / 코드 검색
- `git status`, `git diff`, `git log` 등 읽기 전용 명령
- 탐색적 확인
