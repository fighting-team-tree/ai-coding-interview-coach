# Design Mandate — DESIGN.md 참조 규칙

## 목적
모든 UI/UX 수정 작업 시 `DESIGN.md` 파일을 **반드시** 참조하여 프로젝트의 디자인 시스템과 브랜드 일관성을 유지한다.

## 적용 범위
다음 작업에 해당하는 경우 이 규칙이 **자동으로** 적용된다:
- 컴포넌트 스타일 변경 (색상, 타이포그래피, 간격, 그림자 등)
- 새로운 UI 컴포넌트 생성
- 레이아웃 구조 수정
- 애니메이션 및 트랜지션 추가/변경
- 다크 모드 및 테마 관련 작업
- CSS 변수 또는 디자인 토큰 수정
- 반응형 디자인 브레이크포인트 변경

## 필수 절차

### 1. 작업 시작 전
- `DESIGN.md` 파일을 **반드시 읽고** 현재 디자인 시스템의 원칙, 토큰, 컬러 팔레트, 타이포그래피 규격을 확인한다.
- `.impeccable.md`의 브랜드 페르소나 톤("정확한, 통제된, 자신감 있는")을 함께 참조한다.

### 2. 구현 중
- `DESIGN.md`에 정의된 디자인 토큰(색상, 폰트, 간격 등)을 **우선** 사용한다.
- 새로운 값을 임의로 하드코딩하지 않는다.
- 정의되지 않은 값이 필요한 경우, `DESIGN.md`의 기존 체계와 일관되게 확장하고 사용자에게 보고한다.

### 3. 검증
- UI 변경 결과가 `DESIGN.md`의 "Linear Style" 원칙 (다크 모드 기반, 고정밀 그리드, 투명 테두리 등)에 부합하는지 확인한다.

## 참조 파일 경로
| 파일 | 경로 | 용도 |
|------|------|------|
| DESIGN.md | `./DESIGN.md` (프로젝트 루트) | 디자인 시스템, 토큰, 컴포넌트 가이드 |
| .impeccable.md | `./.impeccable.md` (프로젝트 루트) | 브랜드 페르소나 및 톤 가이드 |
