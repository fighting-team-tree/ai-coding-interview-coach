# AI_CHAMPION 프로젝트 컨텍스트

이 파일은 AI_CHAMPION(코딩 테스트 → 면접 브릿지 플랫폼) 기획과 개발을 보조하기 위한 최우선 참조 룰입니다. 에이전트는 코드 작성, 아키텍처 설계, 기획 제안 시 반드시 본 기준을 준수해야 합니다.

## 1. 플랫폼 핵심 목적
사용자가 알고리즘 문제를 푼 직후, 자신이 작성한 코드와 개념에 대해 **LLM 면접관과 대화형 모의 면접(꼬리 질문)**을 진행하여 기술적인 말하기 능력을 키우는 것이 목표입니다.

## 2. 주요 제약 조건 (Latency 최우선)
- **실시간 음성 면접 타겟 지연시간**: 약 1초 (STT 300ms + LLM 500~800ms + TTS 90ms)
- 클라이언트-서버 간 불필요한 직렬성을 최소화하고, LLM의 첫 바이트(First token)를 즉시 TTS로 전달하는 스트리밍 구조를 1순위로 고려합니다.

## 3. 핵심 기술 스택
- **Frontend**: Next.js + React (상태 및 라우팅 관리)
- **Backend**: FastAPI (비동기, 스트리밍 최적화)
- **AI/DB**: LangGraph(면접 상태머신), Claude API(LLM), Deepgram(STT), Cartesia(TTS), LanceDB/PostgreSQL(문제 및 패턴 RAG)

## 4. 기획 및 평가 시스템 반영 의무
- 면접관 응답 시 환각(Hallucination) 방지를 위해 문제 정보, 제약 조건, 기업별 페르소나 컨텍스트를 반드시 프롬프트 내 RAG 형태로 주입한다고 가정합니다.
- 면접 평가는 다음 '3축 기준'에 집중하여 저장 및 UI 시각화를 고려합니다.
  1. **논리 구조** (문제 해결 방향성이 맞는가?)
  2. **기술 정확도** (시간/공간 복잡도, CS 개념에 대한 설명이 맞는가?)
  3. **설명 명확도** (답변이 간결하고 명확하게 전달되는가?)

💡 **절대 규칙**: 단순한 코딩 테스트 플랫폼의 기능을 넘어서서, '면접 준비'로 이어지는 사용자 여정(User Journey) 파이프라인이 끊기지 않는지를 최상위 점검 항목으로 둡니다.
