# Instructions

본 스킬이 호출되면, 반드시 다음 지침에 따라 응답을 구성하십시오.

## 1. 개요 및 요구사항 분석
주어진 요구사항을 바탕으로, 해당 모듈이 AI_CHAMPION 플랫폼의 핵심 목표(면접 브릿지, 1초 레이턴시, 3축 평가 등)에 어떻게 기여하는지 명확히 하고 제약 조건을 분석합니다.

## 2. 시각화 기반 아키텍처 제안 (Mermaid 필수)
구조, 데이터 흐름, 상태 관리를 설명할 때는 반드시 가독성이 높은 **Mermaid 다이어그램 (Sequence, Flowchart, State Diagram 등)**을 최상단에 포함하여 직관적으로 보여주십시오.

## 3. 핵심 설계 요소 (LangGraph 전략 포함)
- **상태 관리 체계 (State Management):** 각 상태 간 전환 규칙(예: 문제 풀이 완료 -> 면접 대기 -> 꼬리 질문) 및 턴(Turn) 관리에 대한 명세.
- **통신 파이프라인 (Data Pipeline):** Next.js ↔ FastAPI ↔ 외부 API(Claude, Deepgram, Cartesia) 간의 비동기/스트리밍 연결 방식 정의.

## 4. 엣지 케이스 및 복원력 (Resilience)
네트워크 지연, 외부 API 타임아웃, 사용자 이탈, STT 소음 유입 등 예상 가능한 3가지 이상의 시스템 붕괴 포인트를 찾아내고, 각각에 대한 방어 로직(Fallback, Timeout Handling)을 설계 방안에 명시하십시오.
