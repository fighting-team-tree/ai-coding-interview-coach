# Socratic Deep-Dive 플랫폼 기획서 (v2.0)

> **LLM 환각 제어 아키텍처 및 B2B 방향성을 결합한 해커톤(AI Champion) 핵심 제출용 기획**
> 버전: 2.0 | 날짜: 2026-04-19

---

## 1. 프로젝트 개요 (Overview)

### 1.1 해결하려는 문제의 본질
"취준생들은 코딩 테스트용 코드는 잘 외워서 풀지만, **자신의 코드를 논리적으로 설명(시간/공간 복잡도, 트레이드오프)하는 면접**에서는 무너진다."
기존 프로그래머스, 백준이 커버하지 못하는 이 '말하기 훈련'의 간극을 LLM의 꼬리질문(Socratic Method)으로 메웁니다.

### 1.2 핵심 전략 (해커톤 포커싱)
- **과감한 쳐내기:** 해커톤의 핵심인 'AI 기술력'을 돋보이게 하기 위해 MVP 단계에서는 오류가 잦은 **음성 대화 파이프라인(STT/TTS)을 텍스트 모드로 축소**하고, 문제 은행을 수백 개 확보하는 대신 **5~10개의 핵심 알고리즘 패턴**에 집중합니다.
- **방어의 핵심:** LLM 관련 서비스가 늘 받는 "가짜 정보를 말하면 어떻게 할 것인가?(Hallucination)"와 "토큰 비용 감당이 되는가?"라는 두 가지 난제를 시스템 아키텍처 수준에서 해결하여 심사위원을 설득합니다.

## 2. 해결된 핵심 난제 1: 신뢰도 (LLM 환각 방어)

LLM에게 자유를 주면 엉뚱한 정답으로 유저를 혼내고(프롬프트 인젝션/환각), 그렇다고 모든 대본을 IF-ELSE로 짜면 챗봇이 지루해집니다. 이를 해결하기 위해 **[하이브리드 가드레일 (RAG 나침반 + 동적 스피커)]** 아키텍처를 도입합니다.

### 2.1 하이브리드 RAG 데이터베이스 설계
문제 DB는 단순한 문제 텍스트가 아니라, LLM을 조종하는 **'조종간(Fact)'** 역할을 하는 JSON 덩어리로 관리됩니다.
```json
{
  "problem_id": "two_pointer_01",
  "optimal_solution": "시간복잡도 O(N), 공간복잡도 O(1)의 Two Pointer 방식",
  "vulnerability_traps": [
    "유저가 이중 for문(O(N^2))을 썼다면 절대 바로 정답을 말하지 말고, N이 10만일 경우 타이밍 이슈가 생기지 않을지 유도 질문할 것",
    "유저가 list.pop(0)을 사용했다면 배열 메모리 복사로 인한 O(N) 오버헤드를 지적할 것"
  ],
  "forbidden_boundaries": "유저의 제출 언어 스펙과 알고리즘 외에 DB나 네트워크 같은 외부 CS 지식을 묻지 말 것"
}
```

### 2.2 다중 에이전트 역할 분리 (LangGraph 활용)
- **나침반 (Fact Checker):** 위 JSON 기반의 **절대 규칙과 팩트**를 LLM의 시스템 프롬프트 컨텍스트에 강제 주입합니다.
- **면접관 (Interviewer):** LLM은 **주입된 팩트(JSON) 안에서만** 유저에게 던질 질문의 말투와 타이밍을 '자연스럽게(Dynamic)' 생성합니다.
*👉 **심사위원 어필 포인트:** "저희 AI는 지식을 상상해서 내뱉지 않습니다. 인간 전문가가 세팅한 함정(Traps) 가이드에 맞춰, 면접관 역할 연기만 즉석으로 수행합니다."*

## 3. 해결된 핵심 난제 2: 사업성 (비즈니스 및 비용 모델)

API 건당 최고 $0.50에 달하는 유지비용을 취준생의 지갑(B2C)에만 의존하는 것은 현실성 없는 비즈니스입니다. 

### 3.1 B2B / B2B2C 납품 모델로 피벗 (Pivot)
- **타겟 고객의 전환:** '취업 성공률'이 곧 회사의 매출로 직결되는 **국비지원 부트캠프(엘리스, 스파르타코딩 등), 대학교 취업지원처**를 핵심 고객(B2B)으로 잡습니다.
- **가치 제안:** "수백 명의 수강생에게 강사가 일일이 기술 모의 면접을 해주는 인건비를 100분의 1로 줄이고 객관적인 데이터(약점 리포트)를 대시보드로 뽑아줍니다."

### 3.2 토큰 비용 최적화 (Tiered Architecture)
- **압박 꼬리질문 (실시간 채팅):** 속도가 빠르고 비용이 극도로 저렴한 경량 모델(예: Claude 3 Haiku, GPT-4o mini 등)을 활용하여 지연속도 1초 내외의 몰입감 넘치는 대화(Ping-pong)를 유지합니다.
- **최종 3축 평가 리포트:** 3~5턴의 대화가 끝난 직후, 전체 대화 로그를 갈무리하여 **'논리 구조, 기술 정확도, 설명 명확도'를 분석할 때만** 고비용/고성능 추론 모델(Claude 3.5 Sonnet 등)을 단 1회 호출합니다.

## 4. 사용자 여정 시나리오 (Hackathon MVP)

1. **학습 및 제출 (Submit):** 유저가 알고리즘 5대 패턴 중 하나를 선택해 파이썬 웹 에디터로 코드를 작성 후 제출합니다.
2. **분석 및 컨텍스트 주입 (RAG Prep):** 백엔드에서 사용자 코드의 AST(정적 분석) 정보와 해당 문제의 `JSON(Fact/Trap)` 파일을 당겨와 컨텍스트 스페이스를 엽니다.
3. **압박 면접 딥다이브 (Socratic Chat):**
   - AI 면접관 활성화. 유저가 제출한 코드의 허점(Traps)을 기반으로 "이곳 시간복잡도가 어떻게 되죠?", "메모리 최적화 방법은요?" 등의 꼬리질문을 3~4 깊이까지 이어갑니다. 
   - 사용자가 오답을 말하면 해답을 주기보다 스스로 생각하게 넛지(Nudge)합니다.
4. **최종 리포트 생성 (Feedback Assessment):** 면접이 끝나면 고성능 AI가 전체 대화를 평가해 '액셔너블 피드백(Actionable Feedback)' 리포트를 제공하며, 이는 SNS 공유 기능을 포함합니다.
