# 사이트 검증 인터뷰 스크립트

## 목적

- 현재 사이트가 `교육기관/부트캠프` 기준으로 얼마나 설득력 있는지 빠르게 검증한다.
- 핵심 질문은 두 가지다.
  - 이 사이트가 `코딩 테스트 직후 설명 역량을 평가하는 시스템`으로 이해되는가
  - 실제 강사/기관이 도입을 검토할 만큼 신뢰와 운영 가치가 보이는가

## 권장 대상

- 강사/멘토 2명
- 부트캠프 수강생 또는 취준생 2~3명

가능하면 아래 비율을 맞춘다.

- 강사/멘토: 실제 운영 pain 확인
- 수강생/취준생: 사용자 경험과 첫인상 확인

## 진행 시간

- 1인당 10분

## 준비물

- 현재 홈 화면
- 대표 문제 상세 화면
- 대표 문제의 느린 풀이 제출 후 면접 진행 화면
- 최종 3축 리포트 화면
- 기록지 또는 메모 도구

## 진행 순서

### 1. 오프닝 - 30초

아래처럼만 말한다.

> 이 사이트가 무엇으로 보이는지 검증하려고 합니다.  
> 설명은 최소화할 테니, 보이는 그대로 말씀해 주세요.

### 2. 홈 화면 첫인상 - 2분

홈 화면만 30초 보여준 뒤 아래 질문을 묻는다.

1. 이 제품이 무엇인지 한 문장으로 설명해 주세요.
2. 누구를 위한 제품으로 보이나요?
3. 누가 돈을 내고 도입할 제품으로 보이나요?

### 3. 대표 문제 상세 화면 - 3분

대표 문제 화면에서 아래만 보여준다.

- variant 버튼
- 면접 진행 영역
- 질문 근거 영역

질문은 아래로 고정한다.

1. 같은 문제에 다른 코드를 넣으면 질문이 달라진다는 점이 보이나요?
2. 이 화면에서 가장 신뢰 가는 요소는 무엇인가요?
3. 반대로 가장 불안하거나 덜 믿기는 요소는 무엇인가요?

### 4. 최종 리포트 화면 - 2분

3축 리포트까지 보여준 뒤 묻는다.

1. 이 리포트는 개인 연습용으로 보이나요, 기관 운영용으로도 보이나요?
2. 강사라면 이 결과를 어떤 수업/커리큘럼 장면에서 쓰겠나요?
3. 수강생이라면 이 결과를 받은 뒤 무엇을 하게 될 것 같나요?

### 5. 최종 판정 질문 - 2분

마지막은 아래 3개로 끝낸다.

1. 지금 이 사이트를 `실제로 쓸 만한 사이트`라고 보시나요?
2. 그렇지 않다면 가장 먼저 고쳐야 할 한 가지는 무엇인가요?
3. 이 사이트를 한 문장으로 다시 이름 붙인다면 뭐라고 부르겠나요?

## 합성 규칙

인터뷰 3~5건이 끝나면 아래 기준으로 합성한다.

### 유지

- 절반 이상이 홈만 보고 제품을 비슷하게 설명한다.
- 절반 이상이 기관 가치 또는 운영 가치를 자발적으로 언급한다.

### 수정 필요

- 절반 이상이 개인용 AI 면접 앱으로 이해한다.
- 절반 이상이 신뢰 저하 요인으로 fallback, 영문 리포트, 데모 톤을 지적한다.

### 즉시 수정 우선

- 동일 지적이 2회 이상 반복되면 바로 우선순위에 올린다.
- 특히 아래 항목은 한 번만 나와도 중요하게 본다.
  - `누가 돈 내는지 모르겠다`
  - `운영 도구인지 개인 연습 서비스인지 헷갈린다`
  - `임시 데모처럼 보여서 못 믿겠다`

## 인터뷰어 주의사항

- 제품 설명을 길게 하지 않는다.
- “원래는 기관용입니다” 같은 힌트를 먼저 주지 않는다.
- 설득하려 하지 않고, 오해가 어디서 생기는지를 본다.
- 부정 피드백을 막지 않는다.
- 참가자가 쓴 표현을 그대로 기록한다.
