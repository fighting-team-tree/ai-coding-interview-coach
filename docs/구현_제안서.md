## 2. 기술 명세

| 2.1 기술목적 | **비전: "모든 개발자가 자신의 코드를 자신 있게 설명할 수 있는 세상을 만든다."**<br><br>기존 코딩 테스트 플랫폼은 논리적 사고력을 '설명'하는 면접 훈련을 제공하지 못함. 본 기술은 학습자가 작성한 코드를 기반으로 AI가 꼬리질문(Socratic Method)을 4~5 깊이로 던지는 과정을 통해 이 간극을 메움.<br>- **B2B(교육 기관) 가치:** 부트캠프·대학 등에서 반복적으로 발생하는 기술 모의 면접 운영 부담을 줄이고, 수강생별 약점 데이터를 축적할 수 있는 저비용 텍스트 기반 면접 시스템으로 확장 가능<br>- **B2C(개발자) 가치:** 자신이 직접 작성한 코드 기반의 맞춤 압박 면접 훈련을 통해 의사소통 능력, 코드 최적화 센스, 트레이드오프 분석 역량을 종합적으로 향상 |
| --- | --- |
| 2.2 기술구조 | **1. 입력 파이프라인:** 제출된 코드의 경량 정적 분석(Python ast.NodeVisitor 기반: ①중첩 루프 깊이 감지 → 시간복잡도 질문 트리거, ②위험 연산 탐지(list.pop(0), sorted() 등) → Trap 매칭, ③재귀 호출 패턴 인식 → 기저 조건 질문 유도, ④Cyclomatic Complexity ≥15 시 FallbackFlow 자동 분기) 및 해당 문제의 Fact/Trap JSON 데이터 동적 로딩. 코드 채점은 외부 SaaS(Judge0 API)를 활용하여 자체 채점 서버 구축 비용을 줄임<br>**2. 상태 기반 인터뷰 오케스트레이션 엔진:**<br>- Problem Guide: 문제별 optimal solution, follow-up goals, forbidden boundaries, traps를 질문 생성 컨텍스트에 주입하여 면접 질문의 기준점으로 사용<br>- Interviewer(면접관): 주입된 문제 팩트와 코드 프로필 범위 안에서 한 번에 하나의 꼬리질문을 생성하며, 모델 미연동 시에는 규칙 기반 fallback 질문으로 세션을 지속<br>- 상태 분기('Deterministic First' 원칙): AST 코드 프로필 수치 기반 1차 결정 — CC ≥15 또는 파싱 실패 시 FallbackFlow(핵심 개념 질문), 위험 연산 0건 AND 중첩 루프가 낮은 경우 PlanBFlow(스케일업 압박), 그 외 NormalFlow(취약점 기반 꼬리질문). 3가지 경로를 규칙 기반 분기 로직으로 자동 전환<br>**3. 평가 레이어:** 3~4턴의 텍스트 문답 종료 후, 전체 대화 로그를 취합하여 논리 구조·기술 정확도·설명 명확도 기준의 3축 피드백 리포트를 생성하며, 모델 미연동 시에는 fallback 리포트로 대체<br><br>**[구조도 요약]**<br>사용자 코드 제출 → Judge0 채점(또는 Demo Mode) → AST 분석 + Fact/Trap JSON 로딩 → 상태 기반 인터뷰 엔진 (NormalFlow｜FallbackFlow｜PlanBFlow, 3~4턴) → 3축 종합 평가 → 리포트 UI |
| 2.3 주요기능 | **[MVP 핵심 기능]**<br>- 파이썬 기반 알고리즘 문제 해결 및 코드 제출/실행 에디터 (Monaco Editor + Judge0 API)<br>- 제출된 코드상의 취약점 기반 동적 꼬리 질문 및 스케일업(Scale-up) 압박 질문(Plan B) 지원<br>- 면접 종료 후 논리 구조/기술 정확도/설명 명확도 기반 3축 평가 리포트 발급<br>- 턴 수 기반 진행 표시와 텍스트 중심 인터뷰 UI를 통한 면접 완주 경험 제공<br>**[Post-MVP 확장 기능]**<br>- *(신뢰도 확보)* B2B 도입 강사가 AI의 최종 평가를 보정할 수 있는 휴먼 오버라이드(Human-Override) 대시보드<br>- *(확장성 확보)* 강사 전용 커스텀 문항 RAG JSON 자동 생성(Auto-generation) 저작 도구<br>- *(성장 장치)* '논리적 사고력 성장 그래프' 중심 SNS 공유용 피드백 리포트 카드 → B2B2C 바이럴 성장 엔진 |
| 2.4 결과물 형상 | 웹 기반 SaaS 플랫폼 (채용 기관, 교육 기관 타겟 B2B 애플리케이션 형태) |
| 2.5 배포방식 | 반응형 웹 애플리케이션 기반 배포 (프론트/백엔드/데이터 스토어 연동 클라우드 인프라 활용) |
| 2.6 혁신적 요소 | - 프롬프트 자유도를 제약하고 문제별 Fact/Trap 데이터를 컨텍스트로 주입함으로써, 범용 대화형 LLM보다 문제 맥락에 맞는 면접 질문을 안정적으로 생성. AST 분석 결과와 규칙 기반 분기 로직을 결합하여 질문 흐름을 제어하는 구조가 핵심 차별점<br>- **Text-first MVP 기반 안정성 설계:** Monaco 기반 코드 제출, Judge0 연동 또는 Demo Mode, 모델 미연동 시 fallback 질문/리포트로 이어지는 구조를 통해 외부 의존성이 불안정해도 핵심 데모 흐름을 유지하도록 설계 |
| 2.7 도전적 요소 | 비정상적 스파게티 코드 입력 시 AI가 코드 구조를 충분히 해석하지 못할 수 있으므로 발동하는 **'동적 폴백(Fallback) 핵심 개념 질문'**, 반대로 약점이 거의 없는 풀이 입력 시 발동하는 **'시스템 스케일업 압박 질문(Plan B)'** 등 양극단의 엣지 케이스를 하나의 상태 기반 분기 로직 안에서 자연스럽게 처리하는 것이 핵심 난제. 추가로 외부 API(Judge0·LLM) 미연동 상황에서도 Demo Mode와 fallback 질문/리포트로 세션을 계속 진행할 수 있도록 설계하여 시연 중 시스템 안정성을 확보 |

## 3. 구현 방법 및 계획

| 3.1 구현범위 | **세부업무 #1: RAG 나침반(Fact/Trap JSON) 데이터 파이프라인 및 코어 문항(5패턴) 셋업**<br>**세부업무 #2: 경량 AST 파서 + 상태 기반 텍스트 면접 엔진 코어 로직 구현** — NormalFlow, FallbackFlow, PlanBFlow 3가지 상태 분기 포함<br>**세부업무 #3: 면접 평가 프롬프트 엔지니어링 및 3축 피드백 리포트 UI/UX 개발**<br>**세부업무 #4: LLM API Fallback 체계 및 텍스트 중심 인터뷰 UX 통합** — Judge0 채점 연동, Demo Mode, 턴별 진행 UX 구현<br>**[4인 팀 병렬 구조]** ①AI/에이전트 코어, ②데이터/AST/프롬프트, ③프론트엔드 UI/UX, ④인프라/QA — 각 세부업무를 병렬 진행하되 주차별 Go/No-Go 체크포인트(1주차: PoC 성공, 2주차: NormalFlow 동작 확인)로 스코프 리스크를 조기 관리<br>*(확장 로드맵: MVP 이후 ①강사 저작 도구(Trap 자동 생성), ②Human-Override 대시보드, ③SNS 공유 리포트, ④음성 면접(STT/TTS) 파이프라인, ⑤세션 제어 복잡도 증가 시 LangGraph 기반 상태 그래프 오케스트레이션 순차 확장)* |

## 4. 파급효과

| 4.1 기술적 파급효과 | 기존 코딩 교육이 문제 해결 결과(Test cases passed)에만 집착하던 '정적 테스트 중심 패러다임'에서 나아가, 코딩의 의도, 복잡도 판단, 코드 컨텍스트 설명 능력을 묻는 '과정 지향적 Socratic 평가 패러다임'으로 AI 에듀테크 기술을 진일보시킵니다. 시스템에 채택된 제어형(Fact 강제 주입) 구조 모델은 추후 금융, HR 등 다른 B2B 도메인의 상담 훈련 AI 시스템에도 확장이 용이합니다. |
| --- | --- |
| 4.2 사회·산업적 파급효과 | - **비용(산업/확장) 관점:** 부트캠프 등의 B2B 심층 면접 운영 부담을 낮추는 저비용 텍스트 기반 AI 면접 구조를 제시합니다. 특히 타 플랫폼 데이터 크롤링에 의존하지 않고, 강사가 직접 입력한 문항과 내부 면접 데이터를 기반으로 면접 함정(Traps)과 피드백 품질을 점진적으로 개선하는 **'자체 폐쇄형 생태계(Data Flywheel)'**를 구축할 수 있습니다. 이 Flywheel은 ①강사가 문항 입력 → ②AI가 Trap 후보 생성·보조 → ③면접 세션 데이터 축적 → ④AI 질문 정확도 향상 → ⑤더 많은 기관 도입으로 순환하는, 제품 주도 성장(PLG)과 B2B 영업이 결합된 성장 엔진입니다.<br>- **사회 관점:** 글로벌 IT 기업들이 요구하는 '지식을 설명할 줄 아는 소프트웨어 엔지니어링 역량'의 진입 장벽을 대폭 낮춰 교육 정보 격차를 해소합니다. |
