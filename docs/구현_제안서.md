Fact Checker가 규칙 기반 1차 필터(금지 도메인 키워드 블랙리스트, 답 유출 감지) + LLM 보조 2차 판정의 하이브리드 이중 검증으로 바운더리 이탈 시 자동 재생성하며, 대부분의 검증이 결정론적(Unit Testable)으로 처리되어 환각을 시스템 레벨에서 원천 차단<br>**2) 비용 최적화 Tiered Architecture:** 실시간 면접 타임라인에는 극저지연/저비용 경량 모델을 활용하고, 면접 종료 후 종합 평가 시에만 고성능 추론 모델을 1회 호출해 세션당 API 비용 ≤100원의 B2B 사업성(수익성) 모델 확보 (산출 근거: 경량 모델 4턴 면접 ~22원 + 고성능 모델 평가 1회 ~52원 + 채점 API ~2원 ≈ 76원/세션)<br>**3) 코드-컨텍스트 네이티브 면접관:** 기존 AI 면접 툴이 범용 질답에 머무는 반면, 본 기술은 사용자가 직접 작성한 코드의 경량 AST 분석(ast.NodeVisitor 기반 중첩 루프 깊이 감지, 위험 연산 탐지(list.pop(0) 등), 재귀 호출 패턴 인식 + Cyclomatic Complexity 산출) 결과를 구조화된 '코드 프로필 JSON'으로 출력하여 면접 질문을 동적 생성하는 코드 특화 면접 에이전트<br>**4) 전략적 집중(Strategic Focus):** 범용 문제 은행 구축, 완벽한 채점 서버 자체 개발, 실시간 페어 프로그래밍, 음성 대화 파이프라인(STT/TTS)을 MVP 단계에서 의도적으로 배제하고, AI 면접 질문의 '깊이(Depth of Innovation)'에 모든 리소스를 집중하여 텍스트 기반 몰입형 면접 경험의 기술 차별성 극대화 |

## 2. 기술 명세

| 2.1 기술목적 | **비전: "모든 개발자가 자신의 코드를 자신 있게 설명할 수 있는 세상을 만든다."**<br><br>기존 코딩 테스트 플랫폼은 논리적 사고력을 '설명'하는 면접 훈련을 제공하지 못함. 본 기술은 학습자가 작성한 코드를 기반으로 AI가 꼬리질문(Socratic Method)을 4~5 깊이로 던지는 과정을 통해 이 간극을 메움.<br>- **B2B(교육 기관) 가치:** 부트캠프 강사 1인의 기술 면접 시급 약 5만 원 × 수강생 100명 기준 월 500만 원 인건비를 AI 에이전트 건당 약 100원 이하로 대체(검증 완료)하고, 수강생별 객관적 역량 데이터 대시보드를 제공<br>- **B2C(개발자) 가치:** 자신이 직접 작성한 코드 기반의 맞춤 압박 면접 훈련을 통해 의사소통 능력, 코드 최적화 센스, 트레이드오프 분석 역량을 종합적으로 향상 |
| --- | --- |
| 2.2 기술구조 | **1. 입력 파이프라인:** 제출된 코드의 경량 정적 분석(Python ast.NodeVisitor 기반: ①중첩 루프 깊이 감지 → 시간복잡도 질문 트리거, ②위험 연산 탐지(list.pop(0), sorted() 등) → Trap 매칭, ③재귀 호출 패턴 인식 → 기저 조건 질문 유도, ④Cyclomatic Complexity ≥15 시 FallbackFlow 자동 분기. 분석 결과를 구조화된 '코드 프로필 JSON'으로 출력하여 면접관 LLM 프롬프트에 주입) 및 해당 문제의 RAG JSON(Fact/Trap 데이터) 동적 로딩. 코드 채점은 외부 SaaS(Judge0 API)를 활용하여 자체 채점 서버 구축 비용을 제거<br>**2. 다중 에이전트 레이어 (LangGraph State Machine):**<br>- Fact Checker(나침반 — 결정론적 1차 + LLM 보조 2차 하이브리드): 1차 규칙 기반 필터(금지 키워드 블랙리스트 매칭, 답 유출 감지, 중복 질문 체크)로 레이턴시 0ms 즉시 판정 후, 경계선 케이스에만 경량 LLM을 Structured Output(YES/NO)으로 호출하여 forbidden_boundaries 위반 여부를 확정. LLM 검증 호출을 50~70% 절감하면서 Unit Test 가능한 결정론적 검증 확보<br>- Interviewer(면접관): 주입된 팩트의 바운더리 안에서 압박 질문을 유동적으로 생성 (SSE 스트리밍)<br>- 상태 분기('Deterministic First' 원칙): AST 코드 프로필 수치 기반 1차 결정 — CC ≥15 또는 파싱 실패 시 FallbackFlow(핵심 개념 질문), 위험 연산 0건 AND 시간복잡도=최적해 시 PlanBFlow(스케일업 압박), 그 외 NormalFlow(취약점 기반 꼬리질문). 애매한 경계 케이스만 경량 LLM 분류 보조. 3가지 경로를 LangGraph 조건 엣지로 자동 전환<br>**3. 평가 레이어 (Tiered 추론 시스템):** 3~4턴의 텍스트 문답(경량 모델) 종료 후, 전체 대화 로그를 취합하여 고성능 추론 모델 1회 호출로 3축 피드백 추론<br><br>**[구조도 요약]**<br>사용자 코드 제출 → Judge0 채점(Optimistic UI) → AST 분석 + Fact/Trap JSON 로딩 → LangGraph State Machine (NormalFlow｜FallbackFlow｜PlanBFlow, 3~4턴) → 고성능 모델 3축 종합 평가 → 리포트 UI + SNS 공유 카드 생성 |
| 2.3 주요기능 | **[MVP 핵심 기능]**<br>- 파이썬 기반 알고리즘 문제 해결 및 코드 제출/실행 에디터 (Monaco Editor + Judge0 API)<br>- 제출된 코드상의 취약점 기반 동적 꼬리 질문 및 스케일업(Scale-up) 압박 질문(Plan B) 지원<br>- 면접 종료 후 논리 구조/기술 정확도/설명 명확도 기반 3축 평가 리포트 발급<br>- 턴별 진행 표시(Progress Bar) + 힌트 버튼 + 답변 자동 저장을 통한 면접 완주율 극대화 UX<br>**[Post-MVP 확장 기능]**<br>- *(신뢰도 확보)* B2B 도입 강사가 AI의 최종 평가를 보정할 수 있는 휴먼 오버라이드(Human-Override) 대시보드<br>- *(확장성 확보)* 강사 전용 커스텀 문항 RAG JSON 자동 생성(Auto-generation) 저작 도구<br>- *(성장 장치)* '논리적 사고력 성장 그래프' 중심 SNS 공유용 피드백 리포트 카드 → B2B2C 바이럴 성장 엔진 |
| 2.4 결과물 형상 | 웹 기반 SaaS 플랫폼 (채용 기관, 교육 기관 타겟 B2B 애플리케이션 형태) |
| 2.5 배포방식 | 반응형 웹 애플리케이션 기반 배포 (프론트/백엔드/데이터 스토어 연동 클라우드 인프라 활용) |
| 2.6 혁신적 요소 | - 프롬프트 자유도를 제약하고 '조종간(Fact)' JSON을 파라미터화하여 주입시킴으로써 LLM 특유의 할루시네이션(잘못된 지식 창작)을 강력하게 통제. Fact Checker(규칙 기반 + LLM 하이브리드) → Interviewer 이중 게이트 구조로 바운더리 이탈을 시스템 레벨에서 차단. Interviewer의 SSE 스트리밍과 규칙 기반 Fact Check를 병렬 실행하여 이중 검증에도 체감 응답 지연을 단일 LLM 호출 수준(~1.5초)으로 유지하는 Zero-Overhead 검증 아키텍처<br>- **Optimistic UI 기반 제로 레이턴시(Zero Latency) UX:** 코드 제출 직후 외부 채점 API(Judge0)의 실행 결과를 먼저 띄우는 수 초 동안, 백그라운드 비동기(asyncio)로 AST 분석과 Fact/Trap JSON 로딩 및 면접관 인스턴스를 예열하여 유저의 '로딩 이탈'을 방지하고 완벽한 몰입 지원. 자체 채점 서버를 구축하지 않고도 실시간 채점 경험을 제공하는 비용 최적화 설계 |
| 2.7 도전적 요소 | 비정상적 스파게티 코드 입력 시 AI가 예측 맵핑에 실패할 수 있으므로 발동하는 **'동적 폴백(Fallback) 핵심 개념 질문'**, 반대로 약점이 없는 완벽한 복붙 코드 입력 시 발동하는 **'시스템 스케일업 압박 질문(Plan B)'** 등 양극단의 엣지 케이스를 LangGraph 조건 엣지(Conditional Edge)로 끊어짐 없이 처리하는 고난도 아키텍처. 추가로 **외부 LLM API 타임아웃(8초 초과) 시 대체 모델 자동 전환(Retry with Fallback) 및 사전 캐싱된 제네릭 질문 출력**의 3단계 Graceful Degradation과, 외부 API(Judge0·LLM) 장애 시 사전 캐싱된 채점 결과·면접 질문으로 자동 전환하는 Demo Mode를 내장하여 시연 중 시스템 안정성을 보장 |

## 3. 구현 방법 및 계획

| 3.1 구현범위 | **세부업무 #1: RAG 나침반(Fact/Trap JSON) 데이터 파이프라인 및 코어 문항(5패턴) 셋업**<br>**세부업무 #2: 경량 AST 파서 + LangGraph 기반 텍스트 면접관 에이전트 코어 로직 구현** — NormalFlow, FallbackFlow, PlanBFlow 3가지 상태 분기 포함<br>**세부업무 #3: 심의 모델(고비용 모델) 평가 프롬프트 엔지니어링 및 3축 피드백 리포트 UI/UX 개발**<br>**세부업무 #4: LLM API Fallback 체계 및 Optimistic UI 통합** — Judge0 채점 연동, API 타임아웃 Graceful Degradation, 턴별 진행 UX 구현<br>**[4인 팀 병렬 구조]** ①AI/에이전트 코어, ②데이터/AST/프롬프트, ③프론트엔드 UI/UX, ④인프라/QA — 각 세부업무를 병렬 진행하되 주차별 Go/No-Go 체크포인트(1주차: PoC 성공, 2주차: NormalFlow 동작 확인)로 스코프 리스크를 조기 관리<br>*(확장 로드맵: MVP 이후 ①강사 저작 도구(Trap 자동 생성), ②Human-Override 대시보드, ③SNS 공유 리포트, ④음성 면접(STT/TTS) 파이프라인 순차 확장하여 B2B 납품용 문항을 전체 알고리즘 유형으로 확대)* |

| 4.2 사회·산업적 파급효과 | - **비용(산업/확장) 관점:** 부트캠프 등의 B2B 심층 면접 인건비를 획기적으로 절감합니다 (강사 1인 기술 면접 시급 약 5만 원 × 수강생 100명 기준 월 500만 원 → AI 에이전트 API 비용 건당 약 100원 이하로 대체 가능(산출 검증 완료: 경량 모델 면접 + 고성능 모델 평가 1회 ≈ 76원/세션)). 특히 타 플랫폼 데이터 크롤링에 의존하지 않고, 강사가 직접 입력한 문항으로 AI가 면접 함정(Traps)을 자동 추출하는 **'자체 폐쇄형 생태계(Data Flywheel)'**를 구축하여 합법적이고 폭발적인 Scale-up이 가능합니다. 이 Flywheel은 ①강사가 문항 입력 → ②AI가 Trap 자동 추출 → ③면접 세션 데이터 축적 → ④AI 질문 정확도 향상 → ⑤더 많은 기관 도입으로 순환하는, 제품 주도 성장(PLG)과 B2B 영업이 결합된 핵심 성장 엔진입니다.<br>- **사회 관점:** 글로벌 IT 기업들이 요구하는 '지식을 설명할 줄 아는 소프트웨어 엔지니어링 역량'의 진입 장벽을 대폭 낮춰 교육 정보 격차를 해소합니다. |
