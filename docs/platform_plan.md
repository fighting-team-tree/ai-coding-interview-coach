# 코테 → 면접 브릿지 플랫폼 기획서 초안

> LLM 기반 개발자 채용 준비 통합 플랫폼
> 버전 0.1.1 | 2026년 기획 초안

---

## 1. 해결하려는 문제

### 1.1 현재 상황

백준 등 기존 코딩 테스트 플랫폼이 서비스를 종료하고 있으며, 현재 시장에는 코딩 테스트와 기술 면접을 하나의 여정으로 연결해주는 플랫폼이 없다.

| 기존 플랫폼 | 강점 | 한계 |
|---|---|---|
| LeetCode / 프로그래머스 | 문제 풀이 환경 | 면접 설명 연습 불가 |
| Pramp / Interviewing.io | 모의 면접 환경 | 코테 개념 연결 없음 |
| 유튜브 / 블로그 | 개념 학습 | 개인화·피드백 없음 |

### 1.2 핵심 갭

**"코드는 맞는데 면접에서 설명을 못 한다"**

코테를 통과해도 면접에서 알고리즘 선택 이유, 시간복잡도 설명, 트레이드오프 논의를 제대로 하지 못해 탈락하는 경우가 빈번하다. LLM 기술의 성숙으로 이 갭을 자동화된 방식으로 메울 수 있는 첫 번째 타이밍이 도래했다.

---

## 2. 솔루션 개요

코딩 테스트 준비부터 기술 면접 시뮬레이션까지, 개발자 채용 준비의 전 과정을 하나의 플랫폼에서 제공한다. 핵심은 문제를 푼 직후 LLM 면접관이 같은 문제 기반으로 실제 면접처럼 꼬리 질문을 던지는 경험이다.

### 2.1 사용자 여정

| 단계 | 기능 | 사용자 행동 | 플랫폼 역할 |
|---|---|---|---|
| 1단계 | 개념 학습 | 패턴/CS 이론 탐색 | LLM 친화 구조화 레퍼런스 제공 |
| 2단계 | 문제 풀이 | 알고리즘 문제 풀기 | 패턴 태그·힌트 연결 |
| 3단계 | 코드 리뷰 | 풀이 제출 후 분석 | 시간복잡도, 개선점 자동 분석 |
| 4단계 | 면접 시뮬 | 음성/텍스트로 면접 응답 | LLM 면접관 꼬리 질문 + 평가 |
| 5단계 | 약점 분석 | 결과 리포트 확인 | 약점 패턴 탐지, 복습 큐 자동 생성 |

### 2.2 핵심 차별화 기능

#### 패턴 → 면접 연결

알고리즘 패턴 페이지에 "이 패턴으로 나오는 면접 질문 유형" 섹션을 구성한다. 코테 학습과 면접 준비가 같은 개념에서 출발한다는 것을 사용자에게 시각적으로 보여준다.

#### LLM 면접관 (핵심)

문제 풀이 후 "면접 시작" 클릭 시, LLM이 해당 문제의 정답 해설, 시간/공간 복잡도 제약, 레퍼런스를 RAG(검색 증강 생성) 기반 컨텍스트로 받아 실제 면접 형식으로 꼬리 질문을 진행한다. 이를 통해 환각(Hallucination) 현상을 방어하고 기술 평가의 정확도를 극대화한다.

- "방금 사용한 Two Pointer의 시간복잡도를 설명해보세요"
- "O(n²) 브루트포스 대비 이 방법이 왜 나은가요?"
- "공간복잡도를 O(1)로 줄일 수 있는 다른 방법이 있을까요?"

#### 음성/텍스트 실시간 대화 및 압박 면접 모드

실제 면접에 가까운 경험을 위해 음성 대화 방식을 지원하되, 도서관/카페 등 환경적 제약을 고려하여 타이머 기반의 **텍스트 채팅 면접 모드**도 동등하게 지원한다. 
또한 LangGraph State Machine을 활용, 일반 모드와 **"압박 면접 모드(4~5 depth의 날카로운 꼬리 질문)"**를 분리하여 심리적 압박감을 재현한다.

#### 면접 답변 평가

세션 종료 후 3축 평가를 제공한다.

- 논리 구조
- 기술 정확도
- 설명 명확도

각 항목별 점수와 모범 답안을 함께 제시하며, 내 답변 스크립트와 AI의 평가 리포트를 링크 형태로 외부에 공유할 수 있는 기능(Share my report)을 제공하여 취준생 커뮤니티 바이럴 마케팅을 유도한다.

#### 기업별 면접 패턴 및 페르소나 (성벽 구축)

단순한 기출 매핑을 넘어 기존 코테 플랫폼이 모방하기 어려운 '면접관 페르소나'를 세분화한다. "이 기업 면접 준비하기" 역방향 탐색과 함께 다음 스타일을 선택할 수 있다.
- **네이버 스타일**: CS/OS, 언어의 동작 원리 등 기초 체력을 깊게 파고드는 질문
- **카카오 스타일**: 문제 해결 과정의 논리와 알고리즘 선택 이유를 묻는 질문
- **토스 스타일**: 실무적인 트레이드오프와 최적화 방안을 묻는 질문

## 4. 기술 스택

### 4.1 음성 파이프라인

전체 레이턴시 목표: STT 300ms + LLM 500~800ms + TTS 90ms = **약 1초 내외**

```
마이크 입력 → Deepgram Nova-3 (STT) → 텍스트
텍스트 → Claude API (면접관 LLM) → 응답 텍스트
응답 텍스트 → Cartesia Sonic (TTS) → 스피커 출력
```

| 컴포넌트 | MVP | 프로덕션 | 비고 |
|---|---|---|---|
| STT | Web Speech API | Deepgram Nova-3 | MVP 무료 / 프로덕션 $0.012~0.024/분 |
| LLM | Claude API | Claude API | 면접관 페르소나 + 문제 컨텍스트 주입 |
| TTS | Cartesia Sonic | Cartesia Sonic | 90ms 레이턴시, 실시간 대화 최적화 |
| 면접 1회 비용 | 약 $0.25~0.35 | 약 $0.37~0.59 | ElevenLabs 대비 약 60~80% 절감 |

**LLM 스트리밍 최적화**: Claude API를 스트리밍 모드로 호출하여, 첫 문장이 완성되는 시점에 즉시 Cartesia TTS에 전달한다. LLM 전체 응답 완료를 기다리지 않아 체감 레이턴시가 절반 이하로 줄어든다.

### 4.2 서비스 스택

- 백엔드: FastAPI
- 프론트엔드: Next.js + React
- 면접 세션 상태 관리: LangGraph state machine (꼬리 질문 depth 제어)
- 콘텐츠 DB: LanceDB 또는 PostgreSQL + 벡터 인덱스
- MCP 서버: 모든 패턴·문제·면접 Q&A를 Claude Code에서 바로 조회 가능하도록 노출

## 5. 콘텐츠 전략

초기에는 문제 없이도 시작 가능하다. 패턴 레퍼런스를 먼저 구축하고, 이후 LLM 자동 생성으로 문제를 채워나간다.

| 단계 | 콘텐츠 | 수급 방식 | 목표량 |
|---|---|---|---|
| 1단계 | 알고리즘 패턴 | 직접 작성 + LLM 보조 | Two Pointer, BFS/DFS, DP 등 50개 |
| 2단계 | CS 개념·자료구조 | 직접 작성 | 정의·복잡도·사용 케이스 100개 |
| 3단계 | 알고리즘 문제 | LLM 패턴 기반 자동 생성 + 검수 | 패턴당 10문제 = 500문제 |
| 4단계 | 기업별 면접 Q&A | 커뮤니티 기여 + 크라우드소싱 | 주요 기업 20개, 각 50문항 |

**LLM 문제 자동 생성 파이프라인**: 패턴 정의 → LLM에게 해당 패턴 기반 문제 생성 요청 (제목, 설명, 예시, 제약조건, 시간복잡도 포함 JSON 형식) → 사람이 검수 → DB 적재.

## 6. MVP 범위

### 포함

- 알고리즘 패턴 레퍼런스 (50개, LLM 친화 Markdown 포맷)
- 기본 문제 풀이 (코드 에디터 + 채점)
- 풀이 후 LLM 텍스트/음성 대화형 면접 (텍스트 모드 완전 지원)
- 면접 답변 3축 평가 및 피드백
- 사용자 히스토리 저장 및 약점 태그
- 내 면접 리포트 커뮤니티 공유 기능 (바이럴 목적)

### MVP 이후

- 실시간 음성 면접 (Deepgram + Cartesia 풀 파이프라인)
- 기업별 면접 패턴 DB
- MCP 서버 공개
- 커뮤니티 문제 기여 시스템

## 7. 미래 비전

LLM 기술이 발전해도 개발자는 알고리즘과 자료구조를 이해해야 LLM과 함께 코딩할 수 있다. 이 플랫폼은 단기 취업 준비 도구를 넘어, **"LLM 시대의 개발자 기초 역량 훈련소"** 로 포지셔닝할 수 있다.

| 시기 | 포지셔닝 | 핵심 가치 |
|---|---|---|
| 현재 (MVP) | 코테 → 면접 브릿지 | 취준생·이직자 코테+면접 통합 준비 |
| 1년 후 | LLM 친화 코딩 레퍼런스 | AI 에이전트가 참조하는 1순위 소스 |
| 3년 후 | 개발자 역량 훈련소 | LLM 시대 개발자 기초 역량 플랫폼 |
