# 제출용 데모 영상 스크립트

## 목표

- 제출 영상은 `2분 40초 ~ 2분 55초`를 목표로 한다.
- 톤은 `차분한 서술형`으로 고정한다.
- 화면 밖에서 해석하거나 주장하지 않고, 실제로 보이는 흐름만 따라간다.
- 자동 생성본과 수동 촬영 가이드는 모두 `tools/competition-video/scenarios/ai-champion-core.json`을 기준으로 맞춘다.

## 한 줄 포지셔닝

`코드를 제출하면 질문 흐름이 시작되고, 세션이 끝나면 3축 피드백으로 정리됩니다.`

## 데모용 코드와 답변 포인트

### 제출 코드

```python
def min_subarray_len(target: int, nums: list[int]) -> int:
    answer = float("inf")
    for i in range(len(nums)):
        running = 0
        for j in range(i, len(nums)):
            running += nums[j]
            if running >= target:
                answer = min(answer, j - i + 1)
                break
    return 0 if answer == float("inf") else answer
```

### 답변 흐름

1. 현재 풀이가 `O(N^2)`인 이유와, 양수 배열이라 `right`를 뒤로 되돌릴 필요가 없다는 점을 설명한다.
2. `running >= target` 이후 `left`를 줄여도 되는 이유를 `양수 배열 불변식`으로 설명한다.
3. 상태 변수는 `left`, `right`, `current sum`이고, 두 포인터가 각각 한 번씩만 움직여 `O(N)`으로 줄어든다고 정리한다.
4. 다시 제출한다면 모든 시작점을 순회하는 구조를 없애고, 조건을 만족하는 동안만 shrink 하는 슬라이딩 윈도우로 바꾸겠다고 마무리한다.

## 수동 촬영 가이드

- 자동 파이프라인을 쓰지 않고 직접 촬영하더라도 프레이밍은 동일하게 맞춘다.
- 코드 제출 직후에는 `질문 시연` 패널과 `질문 근거` 패널이 모두 보이도록 위치를 고정한다.
- 각 답변 전송 직후에는 채팅 로그를 맨 아래로 내려 최신 질문이 보이는지 확인한다.
- 리포트 구간으로 넘어가면 `3축 피드백` 헤딩과 세 축 카드가 한 화면 안에 들어오도록 다시 맞춘다.
- 면접 구간은 “질문을 읽고 바로 답하는 느낌”이 중요하므로, 오래된 첫 질문이 화면에 고정된 상태로 타이핑하지 않는다.
- 자막과 내레이션은 `보이겠습니다`, `증명합니다`, `설득합니다` 같은 발표자 시점 표현을 쓰지 않는다.
- `왜 중요한지`를 해설하기보다 `지금 무슨 질문이 나오고 무엇을 답하는지`만 짧게 따라간다.
